Machine & Sensor Monitoring System · Dual-Database DBMS

A full-stack Industry 4.0 application that manages real-time sensor streams from 6 factory machines using a polyglot persistence architecture — PostgreSQL for structured entity data and MongoDB Atlas for high-velocity time-series logs.

01 · About
The Problem &
The Build

Modern manufacturing facilities operate dozens of machines, each fitted with multiple sensors reporting temperature, vibration, pressure, proximity, and current values continuously. Managing this heterogeneous data efficiently is precisely the challenge this project addresses.

The system demonstrates how structured relational data (machine specs, operator assignments) and high-frequency, semi-structured time-series logs require two distinct database paradigms — a principle called polyglot persistence.

A real-time simulation engine writes to both databases every 5 seconds, generating statistically distinct sensor distributions for Running, Idle, and Fault machine states.
